# GitHub Teams

Teams are the building blocks for managing members in a GitHub organization. They reflect an organization's structure with cascading access permissions and mentions.

To learn more about teams, please see [About teams](https://docs.github.com/en/organizations/organizing-members-into-teams/about-teams).

## Team Membership

GitHub teams are created and synchronized from the Texas A&M University directory. A team should be created every time a unique set of people need to collaborate on a project or repository.

Teams can be requested using the team creation link for an organization from the [Texas A&M University GitHub portal](https://github.cloud.tamu.edu). A NetID login is required to create a team, and an API endpoint will be available in the future for automated and programmatic team creation.

<em>External collaborators (non-Texas A&M University members with no NetID) cannot be added to teams. External collaborators can only be added individually, directly to repositories.</em>

## Managing Team Members

Team owners can add, remove, and manage members and other owners. Teams are managed as Groups in the Texas A&M University directory in Microsoft Entra ID.

A list of teams owned by a user is available on the [Texas A&M University GitHub portal](https://github.cloud.tamu.edu), with each linking directly to its respective member management in the My Groups section of the Microsoft account portal.

Users added in Microsoft Entra ID Groups will only synchronize to their respective GitHub teams if they are also direct members of the organization. They must be instructed and encouraged to join the organization on the [Texas A&M University GitHub portal](https://github.cloud.tamu.edu). After joining the organization, their team member status should synchronize within one hour.

## Course- and Enrollment-based Teams

Options to create automatically-updating teams based on course and section enrollment are actively being explored, with plans to release this as a feature soon. In the meantime, students should be added directly to a repository without using a team until the enrollment-team feature can be provided and full FERPA compliance ensured.
